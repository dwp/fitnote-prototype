Fit Note Prototype Versions

Please find the Fit Note prototype versions currently under test and refinement

The Baseline Prototypes are developed leveraging the questions from Send your fit note for your ESA claim from the current live Beta.

For any questions, please contact David Doherty.

Fit Note Baseline Prototypes

v1 Prototypes

The Baseline v1 Prototypes are designed using the questions and processes from the Beta.

Prototype v1 versions:

  • v1a - follows the "Desktop and laptop" radio button
  • v1b - follows the "Phone and tablet" radio button
v2 Prototypes

The Baseline v2 Prototypes v2 are designed using a newly introduced GDS component, the step-by-step navigation bar to periodically help claimants. This navigation bar is positioned in the first two-thirds of the form.

Prototype v2 versions:

  • v2a - contains no images
  • v2b - contains images from Beta
  • v2c - contains Project Falcon customised images
  • v2d - integrates a Details GDS component to make the step-by-step navigation bar more discrete. Contains check your answers functionality
Please note: to aid quick and agile prototyping v2b and v2c are just programmed to only show the Save and upload your fit note screen.
v3 Prototypes

The Baseline v3 Prototypes are designed using the same step-by-step navigation bar as Prototype v2, but the bar is positioned in the right-hand-side column.

Prototype v3 versions:

  • v3a - contains no images
  • v3b - contains images from Beta
  • v3c - contains Project Falcon customised images
  • v3d - integrates a Details GDS component to make the step-by-step navigation bar more discrete. Contains check your answers functionality
    • Please note: to aid quick and agile prototyping v3b and v3c are just programmed to only show the Save and upload your fit note screen.
v4 Prototype

Prototype v4 is designed with a series of 'nice to have' new pages for User Testing in the future, specifically with an emphasis on enabling GPs and Job Centre staff. It was decided by Team Falcon to focus on these 'nice to haves' later-on in the project. Therefore for UI Testing please refer to v5.

Prototype v4: incorporates new features such as:

  • An image of a Fit Note on the Start Page
  • The addition of a Warning component that states: This service does not accept or process med 10s, isolation notes or letters from medical professionals.
  • The a Details component with aids the user with additional information for Med 10s, isolation notes and letters from medical professionals.
  • The addition of a Details component with an image of an envelope to aid users with Freepost.
  • The addition of a Do you have a fit note? question and image to ensure that the user is using the right service
  • The addition of a Inset text component stating: Please ensure that your file upload format is either PDF, JPEG or PNG. Do not upload a screenshot. Recommended file size is between 500KB and 10MB.
  • The addition of a Details component stating: What is my National Insurance number?
  • The addition of a page stating: How did you get your fit note? Have you discussed a rehabilitation plan to get back to work?.
  • The addition of a page stating: Have you discussed temporary alternative roles with your employer?
  • The addition of a page with a Submit your fit note checkbox.

Fit Note Prototypes for A/B Testing

v5 Prototypes

Prototypes v5 are designed for A/B UI testing on 19th and 20th November 2020.

  • v5a - a GDS-refreshed version of Beta to be used as the Control during A/B Testing.
  • v5b - Incorporates several new features from Prototype v4 (but not all). The overal aim is to test if the user knows what a fit note is visually and if they know how to upload their fit note correctly (with an emphasis on PDF via mobile and, not as a screenshot).

Fit Note Prototype for Typenote Survey Testing

v6 Prototype

Prototype v6 is designed to test various Content and Components for Typenote Survey Testing starting Friday 27th November 2020. Screenshots will be taken from this prototype and used in Team Falcon's Typenote survey to compare against Content and Components from Prototypes v1 - 5.

! Warning The above prototypes are not for production!
